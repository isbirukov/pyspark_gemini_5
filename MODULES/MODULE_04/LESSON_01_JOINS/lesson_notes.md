# Конспект: Joins

## 1. Типы
* `inner`: Пересечение.
* `left`: Левая + совпадения.
* `full`: Всё из обеих.
* `left_anti`: Только то, чего НЕТ справа (вычитание множеств).
* `left_semi`: Только то, что ЕСТЬ справа (фильтр по наличию).

## 2. Проблема дубликатов
Если соединяете по условию `df1.id == df2.id`, в результате будет два поля `id`.
**Решение:** `df1.join(df2, "id")` — схлопывает ключи.

## 3. Broadcast Join
Оптимизация для "Большая таблица + Маленькая таблица".
Избегает дорогого Shuffle.
`df.join(F.broadcast(small_df), ...)`

## 4. Опасность
`CrossJoin` (декартово произведение) — убивает производительность. Избегать, если не уверены на 100%.
